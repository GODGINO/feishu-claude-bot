# Git Commit 规范

提交代码时必须使用以下格式：

```
<type>(<需求名称>): <简短描述>

<详细描述>

feishuId:<飞书id>
```

## type 字段

- `feat`：新功能（feature）
- `fix`：修补 bug
- `docs`：文档（documentation）
- `style`：格式（不影响代码运行的变动）
- `refactor`：重构（即不是新增功能，也不是修改 bug 的代码变动）
- `test`：增加测试

## 字段说明

- **需求名称**：飞书中的需求或 bug 标题
- **简短描述**：简单描述改动项，不超过 50 字符
- **详细描述**：若有则添加，无则删除，详细改动内容，单行不超过 50 字符
- **feishuId**：便于追查对应需求

## 示例

```
feat(用户登录): 添加微信扫码登录功能

新增微信OAuth2.0授权登录
支持自动绑定已有账号

feishuId:ID12345
```
